# LSTM-Based-Fake-new-Classifier

<div align="center"><h1>📰 Fake News Classifier using LSTM</h1><p><strong>A Natural Language Processing (NLP) deep learning project that classifies news articles as "Fake" or "Real" using Long Short-Term Memory (LSTM) networks.</strong></p><!-- Badges --><p><img src="https://www.google.com/search?q=https://img.shields.io/badge/Python-3.8%252B-blue" alt="Python Version" /><img src="https://www.google.com/search?q=https://img.shields.io/badge/TensorFlow-2.x-orange" alt="TensorFlow" /><img src="https://www.google.com/search?q=https://img.shields.io/badge/Keras-Enabled-red" alt="Keras" /><img src="https://www.google.com/search?q=https://img.shields.io/badge/NLTK-NLP-green" alt="NLTK" /></p></div>Note: This project was built following the concepts taught in Krish Naik's NLP tutorial: Day 10 - LSTM Practical Implementation In NLP Application.📝 Project OverviewThe goal of this project is to build a sequential neural network to accurately predict the authenticity of a news article based on its Title. The model handles text data by converting it into word embeddings before passing it through an LSTM layer for sequence processing.Key Highlights:Text Preprocessing: Used NLTK for Stopwords removal and Porter Stemmer for stemming.Word Representation: Applied One-Hot Encoding (Vocabulary size of 5000) and Word Embedding techniques.Deep Learning Architecture: Utilized TensorFlow/Keras to build a model with an Embedding Layer, LSTM layer, and Dropout layers to mitigate overfitting.📊 DatasetThe dataset used for this project is from the Kaggle Fake News competition. It contains the following columns:ColumnDescriptionidUnique id for a news articletitleThe title of a news articleauthorAuthor of the news articletextThe text of the article; could be incompletelabelMarks the article as potentially unreliable (1 = Fake, 0 = Reliable)Note: For the scope of this model, only the title and label columns were utilized for training.🛠️ Tech Stack & LibrariesLanguage: PythonData Manipulation: pandas, numpyNLP Techniques: nltk, Regular Expressions (re)Deep Learning Framework: tensorflow, kerasModel Evaluation: scikit-learn (Confusion Matrix, Classification Report, Accuracy Score)⚙️ WorkflowData Loading & Cleaning:Loaded the train.csv file using Pandas.Dropped NaN values to ensure clean data input.Extracted the title feature as the independent variable (X) and label as the dependent variable (y).Text Preprocessing:Filtered out special characters and symbols using Regular Expressions.Converted all text to lowercase.Removed English stop words.Applied Stemming using NLTK's PorterStemmer to reduce words to their root forms.Sequence Preparation:Converted the cleaned text into One-Hot Representations with a defined vocabulary size of 5,000.Applied Pre-Padding (using pad_sequences) to ensure all input sequences have a fixed length of 20 words.Model Building (LSTM):Built a Sequential model in Keras.Added an Embedding Layer (Dimensions: 40 features).Added a Dropout Layer to prevent overfitting.Added an LSTM Layer with 100 neurons.Added a final Dense Layer with a sigmoid activation function for binary classification (0 or 1).Compiled using adam optimizer and binary_crossentropy loss function.Training & Evaluation:Split the preprocessed data into Training and Testing sets.Trained the model for 10 epochs with a batch size of 64.Evaluated model predictions using a chosen threshold (e.g., > 0.5) to output final classes.Achieved an accuracy score of ~90-91% on the test data.🚀 How to RunClone the
